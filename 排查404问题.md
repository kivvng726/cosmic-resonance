# 排查 GitHub Pages 404 问题

## 🔍 问题诊断

看到 404 错误通常有以下几个原因：

1. **GitHub Pages 还未启用**
2. **分支或文件夹配置不正确**
3. **部署还在进行中（需要等待）**
4. **仓库是 Private（需要升级账户）**

---

## ✅ 解决步骤

### 步骤 1：检查 Pages 是否已启用

1. **访问设置页面**
   - 打开：https://github.com/jojodd77/cosmic-resonance/settings/pages

2. **检查配置**
   - 应该看到 "Your site is live at..." 的提示
   - 如果没有，说明 Pages 还未启用

3. **正确配置**
   ```
   Source: Deploy from a branch
   Branch: main
   Folder: / (root)
   ```
   - 点击 **Save**

### 步骤 2：检查仓库可见性

1. **确认仓库是 Public**
   - 访问：https://github.com/jojodd77/cosmic-resonance
   - 查看仓库名称旁边是否有 "Public" 标签
   - 如果是 "Private"，需要：
     - 升级到 GitHub Pro（付费）
     - 或改为 Public 仓库

2. **改为 Public（如果需要）**
   - 进入 Settings → General
   - 滚动到底部 "Danger Zone"
   - 点击 "Change visibility"
   - 选择 "Make public"

### 步骤 3：检查 Actions 部署状态

1. **查看 Actions**
   - 访问：https://github.com/jojodd77/cosmic-resonance/actions
   - 查看是否有 "pages build and deployment" 的 workflow
   - 如果显示黄色（进行中），需要等待
   - 如果显示绿色（成功），说明部署完成

2. **如果 Actions 失败**
   - 点击失败的 workflow 查看错误信息
   - 检查是否有构建错误

### 步骤 4：等待部署完成

- **首次部署**：通常需要 1-5 分钟
- **更新部署**：通常需要 1-2 分钟
- **检查方法**：
  - 刷新 Pages 设置页面
  - 查看是否有绿色勾号 ✅
  - 查看 Actions 标签页

---

## 🔧 常见问题解决

### 问题 1：找不到 Pages 设置选项

**原因**：仓库可能是 Private 且账户是免费版

**解决**：
- 将仓库改为 Public
- 或升级到 GitHub Pro

### 问题 2：保存后仍然 404

**可能原因**：
1. 等待时间不够（需要 1-5 分钟）
2. 分支名称不对（应该是 `main`，不是 `master`）
3. 根目录没有 `index.html`

**解决**：
1. 等待 5 分钟后重试
2. 确认分支是 `main`
3. 确认 `index.html` 在根目录

### 问题 3：Actions 显示失败

**检查**：
- 点击失败的 workflow 查看错误
- 常见错误：
  - 找不到 `index.html`
  - 构建超时
  - 权限问题

**解决**：
- 确保 `index.html` 在根目录
- 检查文件是否已提交到 GitHub
- 重新推送代码

在访问网站前，确认：
